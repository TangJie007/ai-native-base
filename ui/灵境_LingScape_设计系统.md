# 灵境 (LingScape) UI 设计系统

> **设计哲学**：晨露 — 清新、通透、流动、有温度
> **版本**：V2.0 — 「晨露」亮色主题
> **日期**：2026-06-12
> **设计师**：UI Designer

---

## 1. 设计理念

### 1.1 "晨露"

灵境的界面像初春清晨的第一缕光——白里透一丝清新绿，透着生命力和创造力。

| 关键词 | 设计表达 |
|--------|---------|
| **清新** | 白底透绿，如晨露映叶，干净透气不压抑 |
| **通透** | 多层纵深用明度差表达，纯白卡片浮在微绿背景上 |
| **流动** | 过渡动画流畅自然，极光边框在亮色下更饱和更鲜活 |
| **温度** | 暖琥珀色点缀，人性化文案，创造力的温暖感 |

### 1.2 视觉签名

**Aurora Border（极光边框）**：核心交互元素在获得焦点时，边框呈现流动的极光色彩渐变。在亮色主题下，极光渐变更饱和更深（oklch 50% vs 暗色版 62%），确保在白底上清晰可见。

### 1.3 灵感来源

- 春日晨露 — 透绿的白，生命力
- 北极光 — 极光边框的直接灵感
- 和纸质感 — 纯白卡片像手漉和纸
- 传统山水画 — "远山如黛，近水含烟"的留白

### 1.4 为什么从暗色切换到亮色

暗色主题的问题：
- 所有竞品（Wallpaper Engine、飞火壁纸等）都是暗色——没有差异化
- 暗色容易让用户觉得"这只是又一个壁纸工具"
- 对于 AI 创作类产品，亮色传递创造力和生命力

亮色主题的优势：
- **白里透绿** = 清新、创造力、生长感——与 AI "创造"主题契合
- 与竞品形成真正的视觉差异化
- 白天使用更舒适

### 1.5 绝对不做

| ❌ 不做 | 原因 |
|---------|------|
| 绿色太浓 | 像未保存的 Word 文档，伤眼 |
| 纯白 #ffffff 作为背景 | 不自然，缺少灵境的"绿意" |
| 紫蓝霓虹渐变 | AI 产品的刻板印象 |
| 毛玻璃滥用 | 性能差、辨识度低 |
| 纯黑 #000000 文字 | 不自然，真实世界不存在 |

## 2. 色彩系统

### 2.1 基于 OKLCH 的色彩空间

灵境使用 OKLCH 色彩空间定义所有颜色，确保感知均匀性。

### 2.2 「晨露」亮色主题色值

### 2.2 语义色彩

#### 主色 — 翡翠绿 (Emerald #34d399)

与灵叶 Logo 的核心色一致，代表"灵"的生命力。

```
--c-primary:        oklch(70% 0.17 163)    /* 品牌主色 — 翡翠绿 #34d399 */
--c-primary-light:  oklch(88% 0.08 163)    /* 浅翡翠 */
--c-primary-dark:   oklch(50% 0.14 163)    /* 深翡翠 */
--c-primary-surface:oklch(96% 0.02 163)    /* 表面填充 — 极淡翡翠白 */
```

#### 强调色 — 嫩绿 (Lime #a3e635)

与灵叶 Logo 的暖端一致，代表"境"的创造力和晨光。

```
--c-accent:        oklch(75% 0.16 90)      /* 嫩绿强调 #a3e635 */
--c-accent-light:  oklch(90% 0.08 90)      /* 浅嫩绿 */
--c-accent-dark:   oklch(55% 0.12 90)      /* 深嫩绿 */
```

#### 中性色 — 极淡翡翠白

背景几乎纯白，只有最微弱的翡翠绿调。

```
--c-bg-deep:      oklch(98% 0.005 163)     /* 侧边栏 — 极淡翡翠白 */
--c-bg-base:      oklch(99% 0.004 163)     /* 主背景 — 几乎纯白，微透翡翠 ≈ #fafefb */
--c-bg-surface:   #ffffff                   /* 卡片/表面 — 纯白 */
--c-bg-elevated:  oklch(99.5% 0.002 163)   /* 提升层 — 更接近纯白 */
--c-bg-overlay:   #ffffff                   /* 弹层/浮层 */
```

#### 文字色

```
--c-text-primary:  oklch(22% 0.01 250)     /* 主要文字 — 近黑 */
--c-text-secondary:oklch(45% 0.015 250)    /* 次要文字 */
--c-text-tertiary: oklch(65% 0.015 250)    /* 辅助文字/占位符 */
--c-text-inverse:  oklch(98% 0.005 163)    /* 反色文字（用于深色按钮/标签） */
```

#### 边框色

```
--c-border-subtle: oklch(92% 0.005 163)     /* 极淡边框 */
--c-border-default:oklch(86% 0.008 163)     /* 默认边框 */
--c-border-strong: oklch(75% 0.015 163)     /* 强调边框 */
```

#### 语义色

```
--c-success:  oklch(70% 0.17 163)          /* 成功 — 翡翠绿 */
--c-warning:  oklch(75% 0.16 90)           /* 警告 — 嫩绿 */
--c-error:    oklch(55% 0.22 20)           /* 错误 */
--c-info:     oklch(75% 0.12 200)          /* 信息 — 冰青 */
```

### 2.3 Logo 匹配渐变

灵境的渐变色直接取自灵叶 Logo 的色谱：暖金 → 嫩绿 → 翡翠 → 冰青 → 雾青。

```css
/* Logo 匹配渐变 — 嫩绿 → 翡翠 → 冰青 */
background: linear-gradient(135deg, oklch(75% 0.16 90), oklch(70% 0.17 163), oklch(75% 0.12 200));
background-size: 300% 300%;
animation: aurora-shift 4s ease infinite;
```

### 2.4 60-30-10 分配

| 比例 | 角色 | 颜色 |
|------|------|------|
| **60%** | 中性背景 | 极淡翡翠白（几乎纯白），纯白卡片 |
| **30%** | 次要元素 | 浅灰边框、次要文字、卡片阴影 |
| **10%** | 强调 | 翡翠绿（交互）、嫩绿（重点标注/渐变起点） |

## 3. 字体排版

### 3.1 字体选择

| 用途 | 字体 | 理由 |
|------|------|------|
| **品牌展示** | 得意黑 (Smiley Sans) | 几何斜切，极具辨识度，现代感 |
| **英文品牌/标题** | Space Grotesk | 几何无衬线，字符间距宽，有"空间感" |
| **中文标题/正文** | 霞鹜文楷 (LXGW WenKai) | 温润书卷气，契合"灵境"意境 |
| **英文正文/UI** | Inter | 屏显优化极佳，小字号清晰 |
| **等宽** | Cascadia Code | Windows 原生等宽字体，用于 API Key 显示和 prompt 代码 |

```css
--font-display: 'Plus Jakarta Sans', 'Noto Sans SC', system-ui, sans-serif;
--font-body:    'Noto Sans SC', 'Plus Jakarta Sans', system-ui, sans-serif;
--font-mono:    'Cascadia Code', 'JetBrains Mono', 'Fira Code', monospace;
```

### 3.2 字号系统

基于 1.25 比例（Major Third）的 5 级字号系统，加上 xs 用于辅助信息。

| Token | 大小 | 用途 |
|-------|------|------|
| `--text-xs` | 0.6875rem (11px) | 标签、辅助信息、元数据 |
| `--text-sm` | 0.8125rem (13px) | 次要 UI 文字、导航项、描述 |
| `--text-base` | 0.9375rem (15px) | 正文、输入框文字 |
| `--text-lg` | 1.125rem (18px) | 小标题 |
| `--text-xl` | 1.375rem (22px) | 区块标题 |
| `--text-2xl` | 1.75rem (28px) | 页面主标题 |
| `--text-3xl` | 2.25rem (36px) | 向导大标题 |

### 3.3 字重

仅使用 4 级字重，保持克制：

| Token | 值 | 用途 |
|-------|-----|------|
| `--weight-light` | 300 | 装饰性文字 |
| `--weight-regular` | 400 | 正文 |
| `--weight-medium` | 500 | 强调文字 |
| `--weight-semibold` | 600 | 标题、按钮 |

### 3.4 行高

| Token | 值 | 用途 |
|-------|-----|------|
| `--leading-tight` | 1.3 | 标题 |
| `--leading-normal` | 1.6 | 正文 |
| `--leading-relaxed` | 1.8 | 长文本、输入框 |

### 3.5 暗色模式排版调整

暗色背景下文字视觉重量更轻，需要：
- 正文行高比亮色模式增加 0.05–0.1
- 正文避免使用 600 以上字重
- 标题与正文的字重对比通过 500 vs 400 实现，而非 700 vs 400

## 4. 间距系统

### 4.1 基于 4pt 的间距

```css
--sp-1:  0.25rem;  /* 4px */
--sp-2:  0.5rem;   /* 8px */
--sp-3:  0.75rem;  /* 12px */
--sp-4:  1rem;     /* 16px */
--sp-5:  1.25rem;  /* 20px */
--sp-6:  1.5rem;   /* 24px */
--sp-8:  2rem;     /* 32px */
--sp-10: 2.5rem;   /* 40px */
--sp-12: 3rem;     /* 48px */
--sp-16: 4rem;     /* 64px */
```

### 4.2 间距节奏

| 场景 | 间距 |
|------|------|
| 页面标题 → 内容 | `--sp-6` |
| 区块之间 | `--sp-6` ~ `--sp-8` |
| 区块标题 → 内容 | `--sp-3` |
| 同组元素间 | `--sp-2` ~ `--sp-3` |
| 卡片内边距 | `--sp-4` ~ `--sp-5` |
| 输入框内边距 | `--sp-2` ~ `--sp-3` |

### 4.3 层次通过间距表达

间距的**变化**创造节奏，不是所有间距都一样大：
- 标题上方留白 > 标题下方留白（拉开标题与上一节的距离）
- 不同区块间距 > 同一区块内元素间距
- 首屏内容上方不留白，直接展示

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 6px | 小元素：标签、小按钮 |
| `--radius-md` | 10px | 输入框、下拉框、卡片内小组件 |
| `--radius-lg` | 14px | 卡片、面板、大按钮 |
| `--radius-xl` | 20px | 向导卡片、弹窗 |
| `--radius-full` | 9999px | 胶囊按钮、状态徽章、进度条 |

## 6. 阴影与纵深

### 6.1 暗色模式下的深度表达

暗色模式下不使用阴影表达深度（几乎看不见），改用**表面色明度递增**：

| 层级 | Token | 明度 | 用途 |
|------|-------|------|------|
| 0 | `--c-bg-deep` | 11% | 侧边栏、标题栏、状态栏 |
| 1 | `--c-bg-base` | 14% | 主内容区背景 |
| 2 | `--c-bg-surface` | 18% | 卡片、面板 |
| 3 | `--c-bg-elevated` | 22% | 输入框、按钮、浮层 |
| 4 | `--c-bg-overlay` | 26% | 弹窗、对话框 |

### 6.2 阴影（仅用于悬浮元素）

```css
--shadow-sm: 0 1px 2px oklch(0% 0 0 / 0.2);
--shadow-md: 0 4px 12px oklch(0% 0 0 / 0.25);
--shadow-lg: 0 8px 24px oklch(0% 0 0 / 0.3);
```

仅用于 hover 时的提升效果，不用于默认状态。

## 7. 动效系统

### 7.1 缓动函数

```css
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);    /* 通用出场 */
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);     /* 弹性出场 */
```

**绝对不使用**回弹/弹性缓动（bounce/spring）——与灵境的空灵气质不符。

### 7.2 时长

| Token | 值 | 用途 |
|-------|-----|------|
| `--duration-fast` | 120ms | 颜色变化、背景切换 |
| `--duration-normal` | 250ms | 状态过渡、展开收起 |
| `--duration-slow` | 400ms | 页面切换、大型动画 |

### 7.3 关键动效

| 动效 | 触发 | 表现 |
|------|------|------|
| **极光边框** | 输入框获得焦点 | 边框呈现流动的青→蓝→金渐变，4s 循环 |
| **呼吸脉冲** | API 连接状态 | 绿色圆点 2s 呼吸式明暗交替 |
| **悬停提升** | 卡片/列表项 hover | translateY(-2px) + shadow-lg，250ms |
| **生成旋转** | AI 生成中 | 双层反向旋转圆环，外环 1s，内环 1.5s |
| **粒子背景** | 全局 | 60个微粒缓慢漂移，鼠标靠近时微弱吸引 |

### 7.4 减弱动效

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 8. 组件规范

### 8.1 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| **主按钮** | 翡翠青渐变填充，白色文字 | 生成壁纸、确认操作 |
| **次按钮** | 透明背景，边框 | 测试连接、取消 |
| **小按钮** | 紧凑尺寸，边框 | 标签内操作 |
| **危险按钮** | 红色边框+红字 | 清除缓存 |

**交互规格**：
- 主按钮 hover：`translateY(-1px)` + `box-shadow: 0 4px 16px oklch(62% 0.18 178 / 0.3)`
- 主按钮 active：`translateY(0)`
- 次按钮 hover：边框加深 + 文字变亮
- 所有按钮 focus-visible：2px 外发光轮廓

### 8.2 输入框

- 背景：`--c-bg-base`
- 边框：1px `--c-border-subtle`
- 圆角：`--radius-md`
- 焦点：边框变 `--c-primary`，外部 Aurora 边框特效
- 占位符：`--c-text-tertiary`（需满足 4.5:1 对比度）

### 8.3 卡片

- 背景：`--c-bg-surface`
- 边框：1px `--c-border-default`
- 圆角：`--radius-lg`
- 内边距：`--sp-4` ~ `--sp-5`
- hover：`translateY(-2px)` + `--shadow-lg`

**禁止卡片嵌套卡片**——用间距和排版表达内部层次。

### 8.4 导航项

- 默认：透明背景，`--c-text-secondary`
- hover：`--c-bg-surface` 背景，文字变亮
- active：`--c-primary-surface` 背景，左侧 3px 主色竖条，文字 `--c-primary-light`

### 8.5 状态徽章

| 状态 | 背景 | 文字 |
|------|------|------|
| 已连接 | `oklch(22% 0.04 155)` | `--c-success` |
| 断开 | `oklch(22% 0.03 20)` | `--c-error` |
| 未配置 | `--c-bg-elevated` | `--c-text-tertiary` |

### 8.6 开关 (Toggle)

- 尺寸：40×22px
- 关闭：`--c-bg-elevated` 背景 + `--c-border-default` 边框
- 开启：`--c-primary` 背景 + `--c-primary-dark` 边框
- 滑块：16px 白色圆，移动 18px
- 过渡：`--duration-normal` `--ease-out-expo`

### 8.7 Toast 通知

- 位置：右上角，距顶 `--sp-4`，距右 `--sp-4`
- 圆角：`--radius-md`
- 包含：图标 + 文字 + 操作按钮（如有）
- 自动消失：4s
- 类型：success / error / warning / info

## 10. 页面设计详述

### 10.1 AI 创作页（核心页面）

**信息架构**（从上到下）：

1. **页面标题** — "AI 创作" + 副标题
2. **描述输入区** — textarea + 参考图上传 + 随机灵感 + 历史记录
3. **风格 & 模型配置** — 风格胶囊按钮 + 模型下拉 + 变体数量 + 分辨率
4. **构建方案预览**（第一步完成后）— 优化后的 prompt + 风格标签 + 色调色板
5. **生成结果**（第二步完成后）— 3 张变体网格 + 设为桌面/保存

**状态流转**：

```
空状态 → 输入中 → 第一步分析中 → 方案展示 → 第二步生成中 → 结果展示
```

### 10.2 壁纸库页

- 顶部：标题 + 筛选标签（全部/AI生成/本地导入/视频）
- 主体：响应式网格，卡片包含缩略图 + 标题 + 元信息
- 当前使用的壁纸有"使用中"标签
- 视频壁纸有"▶ 视频"标签

### 10.3 API 管理页

- 火山引擎卡片（已配置）：显示状态徽章、Key 输入框、模型选择
- 后续平台卡片（半透明/未激活）：显示"V1.1"/"V1.2"标签

### 10.4 设置页

- 分区组织：通用 / 壁纸 / 性能 / 存储 / 关于
- 每个设置行：左侧信息（标签+描述）+ 右侧控件
- 开关使用自定义 Toggle 组件

### 10.5 首次启动向导

**3 步流程**：

1. **欢迎** — 品牌视觉 + 价值主张 + "开始配置"/"跳过"
2. **API Key 配置** — 分步引导 + 输入框 + 测试连接
3. **快速上手** — 3 个预设描述卡片，一键体验

**向导设计原则**：
- 每步一屏，不滚动
- 可跳过，进入本地模式
- API Key 配置有"测试连接"按钮，看到绿灯才能建立信任
- 进度指示器：3 条横线，当前高亮

## 12. 无障碍 (WCAG AA)

### 12.1 对比度

| 元素 | 要求 | 实现 |
|------|------|------|
| 正文文字 | ≥ 4.5:1 | `oklch(92% 0.01 250)` on `oklch(14% 0.018 250)` ≈ 12:1 ✅ |
| 次要文字 | ≥ 4.5:1 | `oklch(68% 0.015 250)` on `oklch(14% 0.018 250)` ≈ 5.2:1 ✅ |
| 辅助文字 | ≥ 4.5:1 | `oklch(48% 0.015 250)` on `oklch(14% 0.018 250)` ≈ 2.8:1 ⚠️ 仅用于非关键信息 |
| 交互元素 | ≥ 3:1 | 按钮和控件边框对比度 > 3:1 ✅ |

### 12.2 键盘导航

- Tab 键按逻辑顺序遍历所有交互元素
- focus-visible 状态：2px 主色轮廓 + 2px 偏移
- 侧边栏导航支持方向键切换

### 12.3 其他

- 触摸目标 ≥ 44px
- 尊重 `prefers-reduced-motion`
- 语义化 HTML + ARIA 标签
- 支持浏览器 200% 文字缩放
